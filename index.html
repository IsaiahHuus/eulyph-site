// the magic symbols
Two characters that replace thousands of lines
Instead of verbose logic, Eulyph gives you two symbols that let you speak directly to the AI engine.
@

Inference symbol

Tell the AI to read data and extract exactly what you need. No parsing logic, no regex, no loops — just describe what you want.

# Extract price from a messy email
price = raw_email @ "total cost"
> $2,500.00
?

Branching symbol

Ask the AI a logical question and let it decide which path to take — based on common sense, not exhaustive if/then chains.

# Route a support ticket
dept = ticket ? "hw or sw?"
> software

// under the hood
A custom engine, built from scratch
Four components work together to make Eulyph's fluid-to-rigid translation possible.
01 // LEXER

The reader

Scans raw code, strips whitespace, and identifies special tokens and symbols before anything else runs.

02 // PARSER

The architect

Takes those symbols and builds an Abstract Syntax Tree — a structural blueprint of the program's logic.

03 // EVALUATOR

The engine

Walks through the blueprint step-by-step, running rigid math normally and passing fluid logic to the local AI model.

04 // STDLIB

The security bridge

Requires a mandatory std.stdy() handshake. Hides dangerous raw execution behind safe, clean wrappers.

// faq
Common questions
Why build a new language instead of just adding AI to Python? ❯
Eulyph treats AI as a native part of the language. By baking concepts directly into the syntax with @ and ?, developers write fluid intent rather than rigid mechanical steps — something no add-on can truly replicate.
Is it safe to let an AI run my computer's code? ❯
Yes. A script cannot run without std.stdy() confirming connection to a safe, approved model. Dangerous raw functions are hidden behind secure wrappers to prevent execution of malicious code.
How does Eulyph talk to the underlying computer? ❯
Eulyph handles the fluid ambiguity of human language. When hard math or system-level work is needed, it securely delegates that specific task down to a traditional Python environment through the Standard Library.
