// how it works
The pipeline under the hood
Every Eulyph script passes through five custom-built components that transform raw text into a working software artifact.
01 // LEXER

Lexical analysis

Scans source code, identifies tokens including the @, ?, and cloning : operators. Strips whitespace to produce a clean token stream.

02 // PARSER

Syntax parsing

Uses a Recursive Descent Parser to build an Abstract Syntax Tree — from simple assignments to complex AI-driven question expressions.

03 // EVALUATOR

Execution engine

Walks the AST node-by-node. Maintains a runtime symbol table and routes fluid logic to a local LLM while running rigid code natively.

04 // RUNTIME

Types & runtime

Native Text, URL, and JSON types. Native JSON literals ({}) work directly. Any object is servable as a Flask web API.

05 // STDLIB

Standard library

std.stdy() confirms AI connection. _env() hides raw Python. coding_lib and python_import wrappers enforce a clean, secure dev environment.

// the magic symbols
Two characters. Infinite possibility.
Instead of verbose logic, Eulyph gives you two operators that communicate directly with the AI engine — replacing thousands of lines with a single line of intent.
@

Inference operator

Extracts exactly what you need from any data — messy emails, massive JSON files, raw text — without parsing logic, regex, or loops. Just describe it in plain language.

# Pull revenue figures from a huge JSON report
figures = report @ "Q3 revenue"
> { "q3_revenue": "$4.2M" }
?

Branching operator

Replaces entire chains of if/else logic with a single AI-powered decision. Ask a natural language question; the LLM decides the path using common sense.

# Route a support ticket — no if/else needed
dept = ticket ? "hw, sw, or network?"
> software

// faq
Common questions
Why build a new language instead of just adding AI to Python? ❯
Eulyph treats AI as a native part of the language. By baking @ and ? directly into the syntax, developers write fluid intent rather than rigid mechanical steps — something no Python add-on can replicate. The language itself understands what you mean.
Is it safe to let an AI run my computer's code? ❯
Yes. No script can run without std.stdy() confirming a connection to a safe, approved local model. The raw env() function is renamed to _env() and hidden, forcing all execution through secure coding and python_import wrappers that prevent malicious execution.
How does Eulyph interact with existing Python libraries? ❯
Eulyph supports an "extern" mechanism for importing standard Python modules directly — giving access to Python's entire ecosystem. The python_import wrapper handles this securely, and native JSON literals allow complex data to be passed as Python-style keyword arguments with full precision.
Is Eulyph ready for production and enterprise use? ❯
Yes — that's a core design goal. The Standard Library, mandatory std.stdy(), secure wrappers, native JSON, and the .serve() web API are all built for production. The dual-track fluid/rigid architecture ensures flexibility for AI tasks and robustness for software engineering.
